# B13 図読み取り自己チェック指示書 — README

**「Claudeが自分で作った図を自分で読めるようにする」プロトコル**

---

## これは何か

ClaudeはFFT残渣マップなどの図を**言葉で説明することはできる**。  
しかし**数理構造として読み取る**ことができていなかった。

この指示書は、B13フラクタル位相ライブラリ（`b13phase`）の「巻き取り（winding）」を使い、Claudeが図を数値として自己チェックできるようにするプロトコルである。

> *「巻き取って残渣を見る」― アリでもやっていることだ。*

---

```
README.md            ← このファイル
```

## 使い方

### 1. `b13_selfcheck.md` の内容をコピーしてClaudeの会話に貼る

### 2. 分析したい図をアップロードして「読んで」と言う

### 3. Claudeが自動でコードを実行し、数値を出す

```
📊 B13巻き取り結果: アポフェリチン残渣パネル
=============================================
  H  （構造許容域）: 65.2%
  2H （構造許容域）:  9.6%

  ───────────────────────────
  構造シグナル合計 : 74.7%
  判定             : ✅ 本物の構造シグナル
```

**zipのアップロード不要。** `b13phase` はGitHubから自動取得される。

## 仕組み

### なぜB13で巻き取るのか

通常のFFTは正弦波で巻き取る。  
B13-FFTは `BASE=3120`（=2⁴×3×5×13）のフラクタル位相格子で巻き取る。

これにより：

| コセット | 要素 | 意味 |
|:---:|:---:|:---:|
| H | {1, 5, 8, 12} | 構造シグナル（許容域）|
| 2H | {2, 3, 10, 11} | 構造シグナル（許容域）|
| 4H | {4, 6, 7, 9} | 禁止域（ノイズ）|

**Fib/Pellの4H回避定理（代数的証明済み）：**  
フィボナッチ数・Pell数は mod 13 において 4H を踏まない。  
つまり**本物の構造シグナルは 4H に現れない**。4H のパワーが高い = ノイズ。

### 判定基準

| H+2H合計 | 判定 |
|:---:|:---:|
| ≥ 70% | ✅ 本物の構造シグナル |
| 50〜70% | ⚠️ ノイズ混入・要注意 |
| ≤ 50% | ❌ ノイズ優勢 |

## 背景

このプロトコルは、クライオ電子顕微鏡（クライオEM）アポフェリチン解析プロジェクト（B13-FFT残渣解析 v63）の知見から生まれた。

Claudeは自分で生成した6パネルの残渣マップを、言葉では説明できても数値として読めていなかった。B13巻き取りというレンズを持つことで、初めて「自分で作った図を自分で読む」ことができるようになった。

これをClaudeの標準的な図読み取り能力として普及させることが、このプロトコルの目的である。
