> 論文未満、実装メモ以上。確定済み命題と観測段階の主張を分離して記述する。

## 1. 目的

標準FFTは平面波分解（回転群の作用）に基づく。  
本実装が観測対象とする coneFFT は、フィボナッチ差分演算子 R₆ と双対ノード N による**反射系の再帰構造**に基づく。

FFT が「どの周波数にエネルギーがあるか」を測るのに対し、  
coneFFT observer は「どの cone 構造への応答があるか」を測る。

両者は互いに置換するものではなく、異なる観測軸を持つ。

---

## 2. 観測器としての coneFFT

### 立場の明示

本実装の `coneSpec()` 出力は、厳密な意味での「coneFFT スペクトル変換」ではない。  
正確には：

```
R₆ 応答 → N-tree 合成 → DFT magnitude 計算 → H(θ) 重み付け
```

によるハイブリッド検出器である。係数 `(sS[m] + dS[m]×0.5) × (1 + |H(θ)|×0.3)` はヒューリスティックを含む。

このことは欠点ではなく、**現時点での正直な立場**である。  
UI の名称も `cone spectrum observer` / `R₆/B13 detector` が理論的には正確だが、継続性のため v0.2.1 では `coneFFT` 表記を維持する。

## 3. R₆ / N / Φ_d の定義（確定）

### R₆ 演算子（フィボナッチ差分）

```
R₆^(d) x(p) = x(p+d) − x(p−d) + x(p+2d) − x(p+3d) + x(p+5d) − x(p+8d)
```

- 核 `x(p+d)−x(p−d)` : 対称二点差
- 外側鎖 : フィボナッチ符号交互（+2d, −3d, +5d, −8d）
- 固有値 L_A ≈ −36.507（成長モード）、L_B ≈ 1.507（収縮モード）

### N 演算子（双対ノード）

```
N(a, b) = (a+b, a−b)
```

- 解析方向（外→内）: `a−b` が主役
- 合成方向（内→外）: `a+b` が主役
- 恒等式: `ab = (s²−d²)/4`（s=a+b, d=a−b）← **cone保存則の核**

### Φ_d（層不変量）

```
Φ_d[R₆x] = Σ_p R₆^(d)(x)(p) · R₆^(d)(x)(p+2d) / n
```

- N を通しても不変（代数恒等式による）
- `Φ_d[x] ≠ Φ_d[R₆x]`（R₆ が x の相関を変える）
- FFT Parseval との対比: Parseval は `‖x‖²` 保存、Φ_d は `R₆後の相関` を保存

### H(θ)（R₆ 固有フィルタ応答）

平面波入力 `x(p) = e^{ik·p}` に対し：

```
H(θ) = e^{iθ} − e^{−iθ} + e^{2iθ} − e^{3iθ} + e^{5iθ} − e^{8iθ}
```

- スペクトルピーク: `m = 3, 10`（`3/13 × 2π`）
- ゼロ点: `kd = 0, π, 2π`

## 4. v0.2.1 の測定指標

### DIFF quantification

| 指標 | 定義 | v0.2.1 修正内容 |
|:--|:--|:--|
| max Δ mode | `argmax |dB(cone[i]) − dB(fft[i])|` | バグ修正済み（`mxAbs=-1` で初期化） |
| max Δ dB | 上記の差分値（符号付き） | 同上 |
| cone優位 bins | `dB_cone − dB_fft > 3dB` の bin 数 | — |

**v0.2 のバグ:** `mxD = -Infinity` で初期化し `Math.abs(mxD)` を取ると `Infinity` になり、最初の1回も更新されなかった。v0.2.1 で修正済み。

### B13 band energy

| 指標 | 定義 |
|:--|:--|
| 評価帯域 | 300Hz ±80Hz、1000Hz ±80Hz |
| cone B13 ratio | `bandEnergy(cone, [300,1000]) / totalEnergy(cone)` |
| fft B13 ratio | `bandEnergy(fft, [300,1000]) / totalEnergy(fft)` |
| cone/fft | 上記の比率 |

**座標系の統一（v0.2.1）:** v0.2 では `Math.round(m/13 × nBins)` により bin座標で評価していた。テスト信号 b13 の注入周波数（300Hz, 1000Hz）と評価窓が一致しておらず、指標の意味が曖昧だった。v0.2.1 では Hz座標系に統一。

### Φ_d session statistics

- **平均 μ**: セッション全フレームの Φ_d 平均
- **分散 σ²**: 同上の分散
- **CV (σ/|μ|)**: 変動係数。小さいほど Φ_d が安定（CV < 0.1 が目安）
- **スパークライン**: フレームごとの推移

### false positive guard

- **B13再現率**: B13信号投入時に cone ピークが 300Hz/1000Hz 近傍（±5%）に出た割合
- **noise偽陽性率**: White Noise投入時に cone B13 帯域比 > 10% になった割合

## 5. B13テスト信号の意味

```javascript
// b13 テスト信号
x[i] = sin(2π × 300t) + 0.5×sin(2π × 1000t) + 0.3×sin(2π × 1300t)
     = sin(2π × fBase×3 × t) + 0.5×sin(2π × fBase×10 × t) + 0.3×sin(2π × fBase×13 × t)
```

`fBase = 100Hz` とし、m=3, m=10, m=13 の整数比を持つ成分を注入する。

B13仮説は「R₆ 演算子が m=3, m=10 モードに構造的な強応答を持つ」と主張する。  
テスト信号はこの仮説を直接刺激するよう設計されている。

**注意:** テスト信号が B13再現率を高くするのは設計上当然でもある。  
仮説の本質的な検証は、**任意入力での cone 固有応答パターンの再現性**にある。

## 6. 偽陽性ガードの考え方

「B13でピークが出た」だけでは証拠にならない。  
必要な対比は：

```
B13信号 → cone B13帯域に高応答   （再現性）
Noise   → cone B13帯域に低応答   （選択性）
```

この両方が揃ったとき、cone observer が B13構造に**選択的**に応答していると言える。

現時点での判定基準（暫定）:

| 指標 | 合格基準 |
|:--|:--|
| B13再現率 | ≥ 80% |
| noise偽陽性率 | ≤ 5% |
| B13 cone/fft 比 | ≥ 1.5× |
| Φ_d CV | < 0.1（B13信号時） |

## 7. 現時点の限界

1. **cone spectrum の近似性**: `coneSpec()` は DFT ベースのハイブリッド。純粋な R₆ 再帰スペクトルではない
2. **dftMag() の間引き**: `step=4` による4点間引き計算。FFT側（フル計算）との精度差がDIFFに混入する可能性がある
3. **座標系の非対称性**: FFT側は Hz 軸、cone側は mode index 軸。横軸の意味が異なる
4. **単一フレーム評価**: 時間変化（スペクトログラム）は未実装
5. **1D実装**: R₆ の本体は3D格子の対角方向。現実装は1D近似
